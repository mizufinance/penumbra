<interfaces>
Existing public DTOs currently in `audit.rs`:
- `AuditDetectedRef { height, tx_hash, action_index, output_index, asset_id, is_flagged, flow_type }`
- `AuditScanExport { scan_info, detected }`
- `OrbisAuditEntry { height, tx_hash, action_index, output_index, asset_id, subject, decision, reason }`

Required new pure API contract in `audit_records.rs`:
- `pub struct AuditImportRow { pub audit_status: String, pub is_flagged: bool }`
- `pub enum OrbisImportEligibility { Eligible, Ineligible { reason: String } }`
- `pub fn classify_orbis_import_row(row: Option<AuditImportRow>) -> OrbisImportEligibility`
- `pub struct DetectedRefRowParts { pub height: u64, pub tx_hash: Vec<u8>, pub action_index: u32, pub output_index: u32, pub asset_id: String, pub is_flagged: bool, pub flow_type: String }`
- `pub fn detected_ref_from_row_parts(row: DetectedRefRowParts) -> AuditDetectedRef`

Classification rules to preserve:
- `Some(row)` is `Eligible` only when `row.is_flagged == false` and `row.audit_status` is one of `AUDIT_STATUS_EVIDENCE_VALID`, `AUDIT_STATUS_DECRYPT_FAILED`, or `AUDIT_STATUS_AUDIT_COMPLETE`.
- Any present but ineligible row returns `Ineligible { reason: format!("row is not an evidence-valid unflagged detection: {status}") }`.
- `None` returns `Ineligible { reason: "detected row not found".to_owned() }`.

Existing status constants remain in `scanner/types.rs`: `AUDIT_STATUS_PENDING`, `AUDIT_STATUS_EVIDENCE_VALID`, `AUDIT_STATUS_EVIDENCE_INVALID`, `AUDIT_STATUS_DECRYPT_FAILED`, `AUDIT_STATUS_AUDIT_COMPLETE`.
`scanner/storage.rs` owns `SqliteScannerStore::lock_conn`; do not open SQLite connections from `audit_records.rs`.
</interfaces>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audit import JSON -> audit DTOs | Orbis/imported audit entries are untrusted input before row eligibility checks. |
| scanner SQLite rows -> export DTOs | Durable scanner rows are projected to exported JSON consumed by audit tooling. |
| refactor scope -> deferred security TODOs | Registration authorization and regulated IBC policy are known security issues but outside this refactor. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `audit_records::classify_orbis_import_row` | mitigate | Preserve the current eligible statuses and unflagged requirement; test eligible, flagged, invalid-status, and missing rows. |
| T-01-02 | Information Disclosure | audit export DTO projection | accept | Preserve existing export fields only; no new data sources or secrets are added. |
| T-01-03 | Elevation of Privilege | registration authorization TODOs | transfer | Do not implement or alter registration authorization in this plan; keep it deferred per D-09 and REQUIREMENTS out-of-scope table. |
| T-01-04 | Denial of Service | failure reason strings | accept | This pure module must not add persistence or expand stored failure blobs; persistence remains in `audit.rs`. |
</threat_model>

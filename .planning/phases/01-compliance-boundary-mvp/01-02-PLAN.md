<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external audit entry -> SQLite mutation | Imported Orbis entries can cause row status changes only after eligibility checks. |
| SQLite scanner store -> exported JSON | Stored rows are serialized for audit tooling without adding new fields. |
| pure helpers -> durable audit facade | Pure classification must not bypass the persistence checks that guard status transitions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | `import_orbis_audit_entries` | mitigate | Keep row lookup, eligible status gate, unflagged gate, and failure recording before any audit-complete update. |
| T-01-06 | Repudiation | audit failure recording | mitigate | Preserve `record_evidence_failure_tx` calls and stage strings for rejected import rows. |
| T-01-07 | Information Disclosure | `export_ledger_rows_json` and `export_scan_json` | accept | Refactor must preserve existing exported fields only and must not add private key, upload bundle, or decrypted plaintext fields. |
| T-01-08 | Elevation of Privilege | deferred registration and IBC policy enforcement | transfer | Block scope creep: no task may implement D-09 deferred security TODOs; these remain out of scope for this phase. |
| T-01-09 | Denial of Service | audit evidence failures | accept | This plan preserves existing failure persistence shape and does not add unbounded new storage. |
</threat_model>
